# 品質ハーネス

結果は `quality-harness-result.md` に書きます。

## 必須フォーマット

```markdown
# 品質ハーネス結果

## スコア

- 情報量: /5
- 正確性: /5
- 網羅性: /5
- 初学者向け理解しやすさ: /5
- 実務理解への接続: /5
- クイズ生成適性: /5
- NotebookLM投入適性: /5

## 判定

合格（PASS） / 追加調査が必要（NEED_MORE_RESEARCH） / 書き直しが必要（REWRITE_REQUIRED）

## 不足している情報

- ...

## 追加調査すべき内容

- ...

## 改善アクション

- ...
```

## 採点ガイド

各カテゴリは1から5で採点します。

- 5: 十分に強く、具体的で、出典があり、NotebookLMに投入できる
- 4: 小さな不足はあるが利用できる
- 3: 利用はできるが、明確な不足がある
- 2: 弱い、リスクがある、または浅い
- 1: 利用できない

## 判定ロジック

### 合格（PASS）

次を満たす場合のみ `PASS` を使います。

- すべての採点カテゴリが4以上
- 利用可能な場合、公式情報または一次情報が含まれている
- `sources.md` に明確な出典表示がある
- 重要用語が定義されている
- 初学者向けの説明がある
- 中級者向けまたは実務向けの深掘りがある
- 例と比較がある
- よくある誤解が扱われている
- `04_quiz_seed.md` に、用語問題、比較問題、引っかけ問題、シナリオ問題に十分な材料がある
- 怪しい情報源や古い情報源が主な根拠になっていない

### 追加調査が必要（NEED_MORE_RESEARCH）

次の場合は `NEED_MORE_RESEARCH` を使います。

- いずれかのスコアが3
- 公式情報または一次情報が薄い
- 例、比較、実務シナリオが弱い
- クイズの種になる材料が十分に豊富でない
- 重要な隣接トピックが欠けている

### 書き直しが必要（REWRITE_REQUIRED）

次の場合は `REWRITE_REQUIRED` を使います。

- いずれかのスコアが2以下
- 出典表示が不明確
- 説明が断片的
- 生成ファイルがNotebookLMのソース教材として適していない
- 怪しい情報、古い情報、矛盾した情報が中心になっている

## 改善サイクル

判定が `PASS` でない場合:

1. 不足しているトピックを特定します。
2. 追加調査を行います。
3. `sources.md` を更新します。
4. 影響を受けるNotebookLM用Markdownファイルを更新します。
5. 品質ハーネスを再実行します。

最大3サイクルまで繰り返します。それでも `PASS` にならない場合は、最善版を残し、残存リスクを明確に記録します。
